# VirtualBox Networking: NAT vs Internal Network Troubleshooting

## Overview

While configuring the homelab network, I initially intended to use two network adapters on each virtual machine:

- **Adapter 1:** NAT for Internet access
- **Adapter 2:** Internal Network for isolated VM-to-VM communication

The intended network design was:

| Machine | NAT Network | Internal Homelab Network |
|---|---|---|
| Ubuntu Server | `10.0.2.15` | `10.0.0.10` |
| Ubuntu Desktop | `10.0.2.15` | `10.0.0.20` |
| Kali Linux | `10.0.2.15` | `10.0.0.30` |

The internal network was configured as:

```text
Network Name: Homelab-NAT
Network Range: 10.0.0.0/24
```

## Problem encountered
After configuring Kali, I expected the interfaces to be separated between the NAT network and the internal homelab network.

I ran
`` ip -br addr ``

and got

<img width="627" height="108" alt="image" src="https://github.com/user-attachments/assets/ab18c198-2bc2-42f3-a1f3-01e841d7b5f7" />

Both interfaces were using addresses from the same network, which was unexpected.

Intended config was
```
eth0 → NAT → 10.0.2.15

eth1 → Internal Network → 10.0.0.30
```

Instead, the config was
```
eth0 → NAT → 10.0.2.15

eth1 → NAT → 10.0.2.12
```

### Diagnosing routing table
I ran
`` ip route ``

and got

<img width="597" height="130" alt="image" src="https://github.com/user-attachments/assets/b3cc6837-6294-427d-8367-faab81e27dd7" />

This confirmed that both network interfaces were connected to the same Layer 3 network ``` 10.0.2.0/24 ```

The presence of ```10.0.2.0/24 dev eth1``` showed that eth1 was also connected to the NAT network

Therefore, the issue was not primarily with Kali's IP configuration. The issue was with my VirtualBox network attachment

## Configuring VirtualBox settings
The correct VirtualBox configuration for Kali is:

### Adapter 1
```
Enabled: Yes

Attached to: NAT
```
### Adapter 2
```
Enabled: Yes

Attached to: Nat Network

Name: Homelab-NAT
```
